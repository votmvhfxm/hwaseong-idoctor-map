# Project: 화성 아이닥터 맵 (Hwaseong Kids-Doctor Map)

## Overview
화성시 소아의료 공백을 AI로 예측·시각화해, 정책 담당자에게는 진료기관 배치 근거를, 시민에게는 지금 갈 수 있는 소아 진료기관을 실시간으로 안내하는 웹서비스. 2026년 화성시 시민·공무원 AI 공모전(접수 마감 7.10 17:00, 발표평가 9.17 예정) 출품작.

**탭 구조와 우선순위**: "정책" 탭 / "시민" 탭 두 개의 큰 축으로 나뉘며, 이번 고도화의 포커스는 **정책 탭**이다. 기획서의 2트랙 구조(A. 정책 대시보드 = 메인, B. 시민 안내 서비스 = 편익 전달 레이어) 원칙을 그대로 따른다 — 심사가 "도시데이터 정책제안" 성격이기 때문.

## Tech Stack
- Language: HTML5 / CSS3 / Vanilla JavaScript (ES6+)
- Framework: 없음 — 빌드 스텝 없는 순수 정적 웹앱 (공고 요건: 설치 없이 브라우저 URL 접속만으로 시연 가능해야 함)
- Database: 없음 — 현재는 코드 내 샘플 데이터(js/data.js), 추후 공공데이터 API로 대체
- 지도: Kakao Maps JavaScript API 적용 중 (국내 주소·길찾기 정확도 우선 고려). `js/shared.js`가 SDK를 동적 로드 — 키 미입력 시 SVG 개념도로 폴백, 입력 시 실제 지도로 전환
- AI: Anthropic Claude API (`claude-sonnet-5`, `/v1/messages`) — 시민 탭 자연어 트리아지에 사용 중
- 백엔드: 기본적으로 없음(정적 웹앱 원칙 유지). 예외 1곳 — `api/directions.js`는 Vercel 서버리스 함수로, 카카오모빌리티 REST API 키를 브라우저에 노출하지 않기 위한 프록시. **Vercel에 배포하지 않고 정적 파일만 열람(로컬 프리뷰, `file://` 등)해도 나머지 기능은 전부 정상 동작** — 이동시간만 데모 추정치(`demoDist`)로 조용히 폴백

```
화성_아이닥터맵/

├── index.html          # 진입점만 담당 (기존 MAKING_Idoctor.html 분리·정리 완료)
├── css/
│   └── style.css        # 공통 스타일
├── js/
│   ├── data.js           # zones, clinics, symptoms, EMERGENCY_KEYWORDS 등 공통 데이터
│   ├── geo.js             # 화성시 28개 읍면동 실제 경계 폴리곤(통계청 행정동 경계, raqoon886/Local_HangJeongDong 2021.04 스냅샷) — 좌표 소수점 5자리로 축약
│   ├── shared.js         # 시간 슬라이더, 커버리지 계산, 지도 렌더링, 뷰 전환 레지스트리 등 공통 로직
│   ├── policy.js         # 정책 탭 전용 로직 (아래 기능 목록)
│   └── citizen.js        # 시민 탭 전용 로직 (아래 기능 목록)
├── api/
│   ├── config.js         # Vercel 서버리스 함수 — 카카오맵 JS 키를 환경변수(KAKAO_JS_API_KEY)에서 읽어 브라우저에 내려줌(자동 연결용)
│   └── directions.js     # Vercel 서버리스 함수 — 카카오모빌리티 Directions 프록시(REST 키는 환경변수 KAKAO_REST_API_KEY로만)
├── docs/                # 참고 문서 (화성기획.pdf — 기술 로드맵. 화성_아이닥터맵_데이터계약서.md — 팀원 UI 셸 인수인계 문서, 실제 구현과 다른 점은 문서 상단에 명시. 소개서·공고문은 상위 폴더에 별도 보관)
└── .claude/
    └── launch.json       # 로컬 프리뷰용 정적 서버 설정 (node/python 미설치 환경 대응 PowerShell 서버)
```
> 파일 분리 완료. `policy.js`/`citizen.js`는 서로 직접 참조하지 않고 `window.Shared`(shared.js가 노출)를 통해서만 상태를 주고받는다 — 새 정책 기능은 policy.js에, 공통 로직(지도·시간·커버리지)은 shared.js에 추가할 것. 빌드 도구는 여전히 필요 없음 (그냥 `<script src="js/data.js">` 순서대로 로드).

## 트랙별 기능 목록

### 정책 트랙 (Track A — 메인, 공무원 대상)

| 기능 | 설명 | 상태 |
|---|---|---|
| 소아의료 공백 지도 | 행정동별 영유아 인구 대비 진료 커버리지를 시간대 슬라이더로 시각화. 야간이 될수록 커지는 공백과, 화성시(특히 동탄 신도시 밀집)와 서부·농촌(공백) 간 격차를 한눈에 대비 | 구현됨(개념도 폴백 + 카카오맵 실연동) → 고도화 진행 중 |
| ㄴ 카카오맵 실지도 연동 | 카카오맵 JS SDK를 동적 로드해 zone 카드·병원 마커·등시간대 원을 실제 지도 위에 표시. `api/config.js`(Vercel 환경변수 `KAKAO_JS_API_KEY`)로 배포 환경에서 자동 연결 시도 → 실패 시(로컬 정적 프리뷰 등) 지도 카드의 "지도 연결 설정"에서 사용자가 수동 입력(브라우저 탭 한정, 미저장)하는 방식으로 폴백. 두 경우 모두 키 자체는 코드에 하드코딩하지 않음 | 구현됨(자동/수동 연결 모두 로컬 검증 완료, 실제 타일 렌더링은 유효 키·도메인 등록 필요) |
| ㄴ 등시간대(isochrone) 반경 | 병원 마커 기준 이동시간 반경(15분/30분)을 지도에 오버레이, 정책 뷰에서 토글. SVG 개념도에서는 상대 좌표 원, 카카오맵에서는 `kakao.maps.Circle`로 실좌표·실미터 반경 표시 | 구현됨(데모 추정 반경 — 직선거리 환산치, 실도로망 아님) → 지도 API Directions 연동 시 정밀화 예정 |
| ㄴ 실제 행정경계 SVG/GeoJSON | 화성시 실제 행정동 폴리곤 경계 사용 | 구현됨 — 통계청 통계지리정보서비스 행정동 경계 데이터(원 출처 [vuski/admdongkor](https://github.com/vuski/admdongkor) 2021.04 스냅샷, 배포 [raqoon886/Local_HangJeongDong](https://github.com/raqoon886/Local_HangJeongDong))에서 화성시 28개 읍면동만 추출해 `js/geo.js`로 축약(좌표 소수점 5자리) 후 카카오맵 위에 `kakao.maps.Polygon`으로 표시. 단, 이 앱의 10개 zone은 28개 읍면동을 인접 지역 기준으로 근사 그룹핑한 것 — 공식 행정구역 "권역" 구분 아님(코드 주석·지도 하단 안내문에 명시). 2026.2.1 화성시 일반구(만세·효행·병점·동탄구) 신설은 이 데이터에 미반영(원본이 2021년 스냅샷) — 읍면동 자체의 이름·경계에는 영향 없음 |
| ㄴ 인구밀도 히트맵 그라데이션 | 커버/미커버 플랫 2색 → 밀도 기반 그라데이션 | 구현됨(zone별 영유아 인구비례 그라데이션 — SVG는 fill-opacity, 카카오맵은 zone 카드 배경 rgba choropleth) |
| ㄴ 24시간 커버리지 트렌드 차트 | 슬라이더 위 스파크라인으로 하루 전체 패턴 표시 | 구현됨(정책 뷰 진입 시 노출, 현재 시각 마커 포함) |
| 배치 우선순위 시뮬레이션 | "이 지역에 야간진료 1곳 추가 시 커버리지 +N%p" 형태로 배치 효과를 즉시 계산해 예산·정책 판단 근거 제공 | 구현됨 |
| 취약 동 랭킹 | 공백 우선순위 리스트 | 구현됨 |
| 소아 진료 수요 예측 (AI) | 영유아 인구밀도 × 요일·시간대·계절 패턴 학습 → 시계열/회귀 예측 | 미구현 (로드맵 2단계) |

### 시민 트랙 (Track B — 편익 전달 레이어, 부모 대상)

| 기능 | 설명 | 상태 |
|---|---|---|
| AI 자연어 증상 트리아지 | 부모가 문장으로 증상 입력 → Claude API가 경증/응급 분류 + 근거·행동 안내. 응급 키워드는 하드코딩 체크리스트가 API 응답 없이 즉시 처리(오분류 방지 안전장치) | 구현됨 |
| 실시간 소아진료 안내 | 문 연 소아청소년과·달빛어린이병원을 거리순으로 안내. 이름 검색, 정렬(거리/대기/마감임박), 필터(야간가능/달빛병원/마감임박제외), "닫힌 곳도 보기" 토글, 클릭 시 상세 모달(진료시간·대기·접수방식·연령·주소·전화, 길찾기/전화하기 버튼) 제공 | 구현됨(샘플 데이터) |
| ㄴ 상시 노출 응급 안내 | 증상 선택 여부와 무관하게 위험 신호(호흡곤란·의식저하·경련·청색증)를 항상 상단에 안내 | 구현됨(정적 배너, 기존 chip 선택 시 뜨는 동적 verdict와 별개) |
| 지도 API 연동 | 실제 지도 위에 가까운 소아 진료기관 마커 표시, 현재 열려있는 곳 강조 | 구현됨(카카오맵 키 입력 시 활성화 — js/shared.js가 정책/시민 뷰 공통으로 소유) |
| 이동시간 안내 | 지도 API의 Directions로 차량 이동시간 표시 | 구현됨(Vercel 배포 시) — `api/directions.js`가 카카오모빌리티 REST API를 프록시. 클리닉 목록은 렌더 즉시 데모 추정치(`demoDist`)로 먼저 보여주고, 백그라운드에서 실이동시간을 조회해 도착하면 정렬·표시를 갱신(초록색 "live" 배지). `/api`가 없는 로컬 정적 프리뷰에서는 404 → 조용히 데모 값 유지, 재요청 안 함. 대중교통 이동시간은 미구현(카카오모빌리티는 자동차 경로만 제공) |
| 길찾기 연결 | 지도 API 딥링크로 카카오맵 앱 길찾기로 바로 연결 | 구현됨(`map.kakao.com/link/to/...` 딥링크, API 키 불필요 — 클리닉 리스트의 "길찾기" 버튼) |

## Code Style Rules
- [ ] 커밋 메시지는 한글로 작성
- [ ] 정책 탭 로직은 `js/policy.js`, 시민 탭 로직은 `js/citizen.js`에만 작성 — 서로 직접 참조하지 말고 `js/shared.js`를 통해 공유
- [ ] 모든 함수에 JSDoc 주석 추가 (`/** @param {string} zoneId */` 형태)
- [ ] AI API 키, 지도 API 키는 코드에 하드코딩 금지 — 입력 필드/환경변수로만 처리 (공고 유의사항: 발표자료에 민감정보 포함 금지)
- [ ] 새 기능을 추가하면 이 문서의 "상태" 컬럼을 반드시 업데이트 (구현됨 ↔ 미구현) — 소개서에 적힌 "구현" 표기와 실제 코드가 항상 일치해야 함 (3차 심사 시 소스코드 검증, 허위 기재 시 수상 취소 조항 있음)

## Commands
- `python3 -m http.server 8000` — 로컬 개발 서버 실행 후 `http://localhost:8000` 접속 (python 없는 환경이면 `.claude/static-server.ps1` 참고 — Claude Preview용 PowerShell 정적 서버, `.claude/launch.json`에 등록됨). 이 방식으로는 `api/directions.js`가 동작하지 않음(순수 정적 서버라 서버리스 함수 실행 불가) — 이동시간은 데모 값으로 폴백
- 별도 빌드/테스트 명령 없음 (정적 파일이라 index.html을 브라우저로 직접 열어도 동작)
- `api/directions.js`까지 포함해 로컬에서 실제로 테스트하려면 `vercel dev` 필요(Vercel CLI, `npm i -g vercel`). 배포는 `vercel --prod` 또는 GitHub 연동 시 push마다 자동 배포

## Important Notes
- 출품 형식: 설치 없이 크롬·엣지 최신 버전에서 바로 열리는 단일 URL. 지도 API·AI API 키가 필요하면 심사위원이 별도 설치 없이 쓸 수 있어야 하므로, 데모용 키 입력 UI 또는 심사용 계정을 함께 준비할 것
- 카카오맵 키 발급 시 카카오 디벨로퍼스 콘솔 → 플랫폼 설정에 **실제 배포 URL**을 웹 플랫폼으로 등록해야 지도가 뜸(등록 안 하면 스크립트는 로드돼도 지도가 비어 보임). 로컬 테스트는 `localhost:8000`도 별도 등록 필요
- 카카오맵 JS 키를 Vercel 배포에서 자동 연결하려면 Environment Variables에 `KAKAO_JS_API_KEY`로 등록(`api/config.js`가 읽음) — 이 키는 브라우저 SDK 로드용 공개 키 성격이라 REST 키와 달리 최종적으로 클라이언트에 노출되지만, 그래도 코드에 직접 하드코딩하지 않고 환경변수로 관리. 미설정/실패 시 지도 카드의 수동 입력 UI로 자동 폴백
- 이동시간 안내(`api/directions.js`)를 쓰려면: (1) 카카오 디벨로퍼스에서 **REST API 키**를 별도 발급(지도 JS 키와 다른 키), (2) 카카오모빌리티 "길찾기" 상품을 활성화, (3) Vercel 프로젝트 설정 → Environment Variables에 `KAKAO_REST_API_KEY`로 등록(코드에 절대 하드코딩 금지) 후 재배포. 이 키는 서버 전용이라 사용자 입력 UI가 필요 없음(AI/지도 JS 키와 다른 패턴)
- AI 트리아지의 응급 키워드 하드코딩 체크리스트(`EMERGENCY_KEYWORDS`)는 절대 제거하지 말 것 — API 실패/오분류 시 마지막 안전장치
- 정책 탭이 메인이므로, 신규 기능 개발 시 정책 탭 완성도(히트맵 정밀화, 예측 모델)를 시민 탭보다 우선할 것
- 디자인 톤: 그린(`--primary:#0E6E5A`, 의료/안심) 베이스 + 화성시 브랜드 오렌지(`--warm`, 활력·야간) 포인트. 마스코트 캐릭터는 아직 미정 — 진행 시 이 문서에 추가
- 팀원(윤정빈) 인수인계 UI 셸(`화성_아이닥터맵.html`, 오렌지 베이스 배색)의 컴포넌트 패턴(낮/밤 히어로, 상시 응급 배너, 파인더 툴바, 상세 모달 등)은 병합했지만 **색상 팔레트는 위 그린 베이스를 유지**함 — 팀원 원본의 오렌지 베이스로 전체 리브랜딩을 원하면 별도 요청 필요
- 팀원 UI 셸의 `fetchData()`(로딩/에러/빈 상태 목업)와 개발용 상태 미리보기 바는 실제 데이터가 정적 동기 로드라 적용하지 않음(정직성 원칙 — 실시간이 아닌 걸 실시간처럼 보이게 하지 않음)
- 참고 문서: `docs/화성기획.pdf`(기술 로드맵·기능·API 명세, 이미지 기반 PDF라 텍스트 추출 불가 — 필요 시 내용은 이 문서의 기능 목록에 반영되어 있음). 솔루션 소개서(서식3)·공모전 공고문은 상위 폴더 `화성시 AI 공모전/`에 있음(프로젝트 폴더 밖, 제출용 별도 문서라 여기로 복사하지 않음)
